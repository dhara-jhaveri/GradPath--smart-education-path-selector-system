Write a program to reverse a string without using any built-in functions.

Solution Approach:

To reverse a string without using built-in functions, we can use a two-pointer approach:

// C++ Solution
#include <iostream>
using namespace std;

void reverseString(char* str) {
    // Find the length of the string
    int length = 0;
    while (str[length] != '\0') {
        length++;
    }
    
    // Swap characters from start and end
    int start = 0;
    int end = length - 1;
    while (start < end) {
        char temp = str[start];
        str[start] = str[end];
        str[end] = temp;
        start++;
        end--;
    }
}

int main() {
    char str[100];
    cout << "Enter a string: ";
    cin.getline(str, 100);
    
    reverseString(str);
    cout << "Reversed string: " << str << endl;
    
    return 0;
}

Time Complexity: O(n) where n is the length of the string

Space Complexity: O(1) as we're reversing in-place

Implement a function to find the nth Fibonacci number using recursion.

Solution Approach:

The Fibonacci sequence is defined as: F(0) = 0, F(1) = 1, and F(n) = F(n-1) + F(n-2) for n > 1.

// Java Solution
public class Fibonacci {
    public static int fibonacci(int n) {
        // Base cases
        if (n <= 0) return 0;
        if (n == 1) return 1;
        
        // Recursive case
        return fibonacci(n-1) + fibonacci(n-2);
    }
    
    public static void main(String[] args) {
        int n = 10;
        System.out.println("The " + n + "th Fibonacci number is: " + fibonacci(n));
    }
}

Note: This recursive solution has exponential time complexity. For large values of n, consider using dynamic programming or an iterative approach.

What is the difference between process and thread?

Process:

  • A process is an instance of a program in execution
  • Each process has its own memory space, system resources, and security context
  • Processes are isolated from each other
  • Context switching between processes is more expensive
  • Communication between processes requires inter-process communication (IPC) mechanisms

Thread:

  • A thread is a lightweight process, or a unit of execution within a process
  • Threads within the same process share memory space and resources
  • Threads can directly access shared data within a process
  • Context switching between threads is less expensive
  • Communication between threads is easier as they share memory

In summary, processes provide a higher level of isolation but with more overhead, while threads offer better performance but require careful synchronization when accessing shared resources.

If a train travels at 60 km/hr, how long will it take to cover a distance of 240 km?

Solution:

We can use the formula: Time = Distance / Speed

Time = 240 km / 60 km/hr = 4 hours

Therefore, the train will take 4 hours to cover a distance of 240 km.

Find the next number in the sequence: 2, 6, 12, 20, 30, ...

Solution:

Let's analyze the differences between consecutive terms:

  • 6 - 2 = 4
  • 12 - 6 = 6
  • 20 - 12 = 8
  • 30 - 20 = 10

We can see that the differences form an arithmetic sequence: 4, 6, 8, 10, ...

The next difference would be 12.

Therefore, the next number in the sequence is 30 + 12 = 42.

If 5 workers can complete a task in 12 days, how many days will it take for 8 workers to complete the same task?

Solution:

This is an inverse proportion problem. The time taken is inversely proportional to the number of workers.

Let's denote the number of days for 8 workers as x.

We can set up the proportion: (5 workers × 12 days) = (8 workers × x days)

5 × 12 = 8 × x

60 = 8x

x = 60 ÷ 8 = 7.5 days

Therefore, 8 workers will take 7.5 days to complete the task.
